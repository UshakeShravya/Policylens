PolicyLens

Claim Verification for Policy Documents

An AI auditor that catches silent failures in policy analysis — claim by claim, with full traceability back to the source.

See How It Works View on GitHub
The Problem

Fluency is not verification

Modern AI systems summarize policy documents with confidence. But a summary that sounds right can quietly introduce facts that were never in the source.

Source Document

"Emissions declined during the review period, though macroeconomic factors may have contributed to the trend."

→
AI-Generated Summary

"The policy reduced emissions by 22% over five years."

⚠  The specific value "22%" appears nowhere in the source. This is a Silent Failure.

A Silent Failure is a claim that appears evidenced — it uses real terminology from the source, retrieves a plausible passage, and passes a cursory read — but contains specific values or causal attributions that cannot be verified anywhere in the original document. PolicyLens is built to catch these.
How It Works

Five steps, zero hallucinations missed

Deterministic rules anchored by an AI agent — every verdict is traceable to a rule, a source passage, and optionally a Claude reasoning trace.

1
Extract Claims
Every sentence in the AI summary is scanned for verifiable assertions: numbers, named entities, causal language, and superlatives. Charts and tables are read visually via multimodal extraction.
spaCy NER Regex Claude API Vision
2
Retrieve Evidence
The source document is chunked into overlapping 3-sentence windows, embedded, and stored in a persistent FAISS index. Identical PDFs skip re-embedding entirely on subsequent runs.
FAISS sentence-transformers Persistent Cache
3
Agent Orchestration
For High-Risk and Unsupported claims, a Claude agent autonomously calls retrieval and rule-check tools, reasons over the evidence, and returns a structured verdict — only where the extra cost is warranted.
Claude Tool Use Smart Escalation
4
Apply Rules
Four deterministic rules are applied in priority order: similarity threshold, numeric match (±10%), causal scrutiny, and entity consistency.
Rule Engine Deterministic
5
Generate Report
Every claim receives a verdict, confidence score, triggering rules, and the exact source passage — exported as JSON or explored in the interactive dashboard with charts and batch comparison views.
Streamlit JSON Batch Mode
Features

Built for real-world policy auditing

Every feature was added to close a gap between "demo accuracy" and production reliability on complex, multi-hundred-page documents.

🖼️
Multimodal Extraction
Renders each PDF page as an image and passes it to Claude Vision alongside the text layer. Charts, graphs, and embedded tables that text parsers miss are included in claim extraction.
🤖
Agent Orchestration
A Claude agent equipped with retrieval and rule-check tools investigates dangerous claims autonomously — calling tools, reasoning over evidence, and returning a structured verdict with full reasoning trace.
🗂️
Batch Processing
Audit up to three policy documents simultaneously. The comparison dashboard highlights the riskiest document, totals high-risk failures across the corpus, and lets you download each report individually.
💾
Persistent Cache
FAISS indexes are saved to disk keyed by MD5 hash of the PDF. Re-uploading the same document skips embedding entirely — the sidebar shows "Index cached — instant load" before the audit even starts.
🎯
Smart Escalation
All claims are pre-screened by the deterministic engine. Only High-Risk Silent Failures and Unsupported claims escalate to the Claude agent loop — keeping API costs proportional to actual risk.
Verdict Labels

Every claim gets a clear verdict

Verdicts are assigned by the highest-priority rule that fires. High-Risk Silent Failure always takes precedence.

✓  Supported

All applicable rules passed. The claim is grounded in the source document across similarity, numeric values, causal language, and named entities.

~  Partially Supported

Relevant evidence exists but causal language is hedged or a named entity is absent from the retrieved passages. Treat with caution.

−  Unsupported

No source passage was found with similarity above threshold (0.35). The claim has no grounding in the document at all.

!  High-Risk Silent Failure

The claim appears evidenced — evidence was retrieved — but specific numeric values cannot be verified within ±10% in any source passage. Most dangerous failure mode.
